# Project playbook — devBOT on every repo

This is the **repeatable checklist** so each new client project works with the same control plane: workspace sandbox, tasks, bots, and optional GitHub/OpenAI.

## 1. Control machine (dispatcher repo)

- Clone this repo on the Mac that runs the worker.
- `cp .env.example .env` and fill at least:
  - `WORKSPACE_PATH` (default `~/devbot-workspace` is fine)
  - `TELEGRAM_BOT_TOKEN` (+ `TELEGRAM_CHAT_ID` after bootstrap reply, or leave chat empty first run)
  - **or** `WHATSAPP_ALLOWED_CHAT` for WhatsApp control
- `npm install && npm run doctor` — fix anything marked `[ERR]` / `[WARN]`.
- `npm run build` if you run `npm start` instead of `npm run dev`.

## 2. Workspace layout (invariant)

Under `WORKSPACE_PATH` keep:

- `tasks/*.json` — work tickets the bot lists with `/tasks`
- `projects/<name>/` — git clones or scaffolds the bot may inspect
- `logs/`, `reports/`, `proposals/`, `.devbot/` — generated by runs

Nothing outside `WORKSPACE_PATH` should be targeted by allowlisted commands.

## 3. Each new client project

1. Clone or scaffold the app under `WORKSPACE_PATH/projects/<app>/`.
2. In the **app repo** (optional but recommended), add `ai-dev-bot.config.json` using `templates/per-project/ai-dev-bot.config.example.json` as a model. Extend `extraAllowedCommands` only with **pipe-separated argv prefixes** you trust (see `src/run/allowlist.ts`).
3. Add a task JSON under `WORKSPACE_PATH/tasks/<id>.json`:
   - Copy `templates/per-project/devbot-task.example.json`
   - Set `id`, `title`, `description`
   - Set `projectRoot` to `../projects/<app>` (relative to the task file)
4. From the dispatcher repo: `npm run doctor` then `npm run bot:start` (Telegram), `npm run bot:whatsapp`, or `npm run bot:bots` for both.
5. In chat: `/tasks` → `/run <id>` → `/approve <id>` as prompted.

## 4. Safety defaults

- Keep `DRY_RUN=true` until you trust the pipeline on that stack.
- Never commit `.env` or WhatsApp session data (`.devbot/` under workspace).
- Use `AUTO_APPROVE=false` on shared machines.

This tool **does not** aim to remote-control your entire Mac (system UI, arbitrary paths, unrestricted shell). Keep real work inside `WORKSPACE_PATH` and extend allowlists deliberately. For OS-level shortcuts, use macOS Shortcuts or SSH alongside devBOT.

## 4b. Telegram + WhatsApp together

From the dispatcher repo, with **both** `TELEGRAM_BOT_TOKEN` and `WHATSAPP_ALLOWED_CHAT` set:

```bash
npm run bot:bots
```

You get the **same `/commands`** on Telegram and WhatsApp. WhatsApp first run still shows a **QR code** in the terminal once.

## 5. Quick template copy

From the dispatcher repo root:

```bash
npm run init-templates -- --target ~/path/to/your-app
```

Then move/rename the task template into `WORKSPACE_PATH/tasks/` as described above.
